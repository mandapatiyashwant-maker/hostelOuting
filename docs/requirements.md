# Hostel Outing Management System

## Users

1. Student
2. Academic Mentor
3. Warden
4. Admin

## Workflow

Student applies outing
→ Mentor approval
→ Warden approval
→ Outing pass generated
→ Student returns
→ Outing record updated
→ Mess fee deduction calculated

## Features

### Student
- Login
- Apply outing
- Track status
- Download pass

### Mentor
- Approve/Reject requests

### Warden
- Approve/Reject requests
- Generate pass

### Admin
- Reports
- User management
